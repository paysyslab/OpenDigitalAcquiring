Change/Forgot Login

Change Login Pin - Functional Flow​

Requester Action(s)Process Description
User (Navigation)• Login: User logs into the App.
• Menu Selection: User clicks the Hamburger menu and selects Settings.
User (Selection)• User clicks on the Change Login PIN option.
User (Input)• User is prompted to enter:
  o Current PIN
  o New PIN (Entered twice for confirmation)
• User clicks Continue.
App Service (Validation)• Mobile App calls the SetPIN Method.
• History Check: Validates that the New PIN does not match the Last 5 PINs (Password History).
• Security Action: App Service disables the user's fingerprint/biometric login in the database to enforce security.
App / UI• Success: User is shown the "PIN Changed Successfully" screen.

Forgot Login Pin - Functional Flow​

Requester Action(s)Process Description
User (Initiation)• User is on the Login Screen.
• User clicks the Forgot Login PIN button.
User (Identity Verification)• User is asked to provide:
  o CNIC
  o Mobile Number
  o Date of Birth (DOB)
• User clicks Confirm.
• Note: If details match, proceed. If incorrect, show "Account Not Found".
App Service (Backend)• Calls ForgotPIN method to verify credentials.
• If Verified:
  o Calls /registration/forgotpass/.
  o Security: Disables user's fingerprint/biometrics in the database.
  o OTP Generation: Generates a One-Time Login OTP and sets it as the user's Temporary PIN.
Gateway (Communication)• SMS/Email Gateway: Sends the OTP (Temporary PIN) to the user's registered mobile number/email.
User (Temporary Login)• User enters the PIN received via OTP into the login field.
• App calls authenticate method to verify the temporary credential.
User (PIN Reset)• Upon successful login, user is forced to change the PIN.
• User enters:
  o Old PIN (The OTP received).
  o New PIN (and confirms it).
• User clicks Submit.
App Service (Finalization)• Calls SetLoginPIN method.
• Sets the New PIN as default.
• Shows Success Message to the user.
