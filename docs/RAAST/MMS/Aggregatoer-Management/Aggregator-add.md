# Add / Update Aggregator

## Overview
The **Add Aggregator** functionality allows users to create a new aggregator record in the system by entering required identification, settlement, and configuration details. This screen is used to register aggregators and define their beneficiary settlement information along with operational status.

## Field Details

**Code***
- Unique identifier assigned to the aggregator.
- Used to distinguish the aggregator within the system.

**Name***
- Name of the aggregator entity.

**Beneficiary Account Number***
- Settlement account number associated with the aggregator.

**Beneficiary Name***
- Name of the beneficiary linked with the settlement account.

**Beneficiary Bank***
- Bank where the beneficiary account is maintained.

**Is CAS Enabled***
- Dropdown selection indicating whether CAS is enabled.
- User selects the applicable option from the list.

**Status***
- Dropdown field used to define the aggregator status.
- User selects the required status option.

## Submission

**Submit Button**
- Saves the entered aggregator details.
- Creates the aggregator record in the system.


## Checker Approval

- Once the aggregator details are submitted, the request is routed to the checker for approval.
- The aggregator record becomes effective only after checker approval.
