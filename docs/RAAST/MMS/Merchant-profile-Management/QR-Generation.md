# QR Generation

## Single QR Generation Functionality

The QR Generation feature appears to facilitate the creation of QR codes for individual merchants. Here's how the process works:

**Merchant Selection:** A user selects a merchant from the list, identified by their name (e.g., "Shubash Sinha") and business details (e.g., "Shubash hair saloon").

**Terminal Selection:** The user must select terminals for the merchant from the available options (displayed in the "Terminals" column), which could represent different types of payment devices or systems.

**Acquirer Selection:** The user selects the acquirer(s) for the transaction, such as RAAST, VISA, MasterCard, or PayPak, based on the merchant's payment system.

**QR Generation:** After selecting the relevant information, a user can click to generate the QR code, which will be used for payment processing through the relevant terminal and acquirer.

## Bulk QR Generation Functionality

This feature enables the generation of QR codes for multiple merchants in one go, which is particularly useful for large-scale operations. The process includes:

**Bulk Selection of Merchants:** A user can select multiple merchants from the displayed list, either individually or by selecting all merchants via a checkbox or the "SELECT ALL" button.

**Acquirer and Terminal Configuration:** The user can set the acquirer (e.g., RAAST, VISA, MasterCard) and terminal details that should apply to all selected merchants.

**Bulk QR Generation:** Once the merchants and settings are selected, the user can initiate the bulk QR code generation process. This will create QR codes for each merchant, and these can be downloaded or saved for distribution to customers or for use in transactions.
