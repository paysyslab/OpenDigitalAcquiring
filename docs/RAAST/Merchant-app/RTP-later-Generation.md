# RTP Later Request Generation

## RTP Later Request Setup

- Merchant selects the **Later** option on the Raast RTP screen.
- Merchant enters the Raast ID or IBAN of the customer.
- Payment rail option is selected as displayed.
- Disclaimer message is shown before proceeding.
- Merchant taps **Verify**.

## Amount Entry

- Customer account details are displayed after verification.
- Merchant enters the payment amount using the keypad.
- Entered amount is displayed dynamically.


## Expiry Selection

- Merchant selects expiry duration in **days**.
- Expiry text field available.
- Expiry time must not exceed 40d.
- Merchant can add optional comments.
- Merchant taps **Generate Request**.

<div style={{ width : '30%' }}>
  ![RAAST RTP later](/img/RTP-later/4.png)
</div>

## Request Generated

- Confirmation screen displays generated RTP request.
- Requested amount is shown.
- Remaining validity duration is displayed.
- Merchant taps **OK** to acklaterledge.

<div style={{ width : '30%' }}>
  ![RAAST RTP later](/img/RTP-later/5.png)
</div>

## Expired Request Handling

- If the RTP expires, expiry status is displayed.
- Merchant can regenerate a new RTP request.
